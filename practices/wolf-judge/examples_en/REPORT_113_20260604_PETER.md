## Design Decisions

- **No game_id on audit_log**: audit logs are "operation auditing" not "game replay", distinct from HistoryService's game_id-linked history tables.
- **Silent failure**: all auto-recording points use the `log_audit` helper; failure does not return an error or block the main flow.
- **timestamp_ms uses SystemTime**: complies with TASK constraint (millisecond, `std::time::SystemTime`).
- **resolve_night death loop**: multiple deaths possible at night; each death gets its own audit log entry with cause of death in details.
