# DECISION_002: 最终产物只有 TASK 和 TODO 两种

**结对**: @tpm-pair (Reasonix + Zehee)
**时间**: 2026-06-09 22:15

---

## 决策

AgentCharter 协作系统的最终产物只有两种：TASK（包含代码变更和文档变更）和 TODO（排期事项）。DOCS 不是独立产物类型，文档变更属于 TASK 的一种内容形式。

## 推理链

- **Zehee**: "我们确定好整个协作系统的最终产物是什么，目前我们定义了三种，task、todo 和 doc。好像 doc 的更改也需要 task，所以 doc 更改不是最终产物，只是 task 的具体内容，应该并入 task。"
- **Reasonix**: "对。代码、文档、配置——所有变更都走 TASK。DOCS 不是独立产物类型，它是 TASK 的内容。"

## 替代方案

1. DOCS 作为独立产物类型 → 导致双重追踪（既有 TASK 又有 DOCS 记录同一个文档变更），否决
