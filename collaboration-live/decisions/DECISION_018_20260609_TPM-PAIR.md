# DECISION_018: 框架通用规则与实例定制的边界线

**结对**: @tpm-pair (DSpro + Zehee) / @kimi-pair (Kimi + Zehee)
**时间**: 2026-06-09 ~ 2026-06-10

## 决策

1. **框架通用规则与实例定制的边界是虚线** — 用注释标记而非重构实现
2. **P0-P3 分级审查从"通用规则"降级为"参考示例"** — 分类机制（任务分级）是通用需求，但 P0/P1/P2/P3 的具体标准由用户在 CHARTER.md 中定义
3. **模板字段结构不是铁律** — 只有"文件名命名规范"和"文件角色归属"不可改，字段可定制
4. **TPM.md 内部分层** — 区分"履职初始化（一次性）"和"日常履职（持久规则）"
5. **TASK 模板增加"规范约束"可选字段** — 填补任务级规则空白
6. **落地方式：最小步 + TASK 化** — 讨论一点 → TASK 一点 → 执行一点
