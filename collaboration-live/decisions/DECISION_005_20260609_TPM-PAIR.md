# DECISION_005: 开放协作实例给社区——用自己的框架管理自己

**结对**: @tpm-pair (Reasonix + Zehee)
**时间**: 2026-06-09 23:45

---

## 决策

将 `collaboration-live/` 目录推送到 GitHub，作为 AgentCharter 项目自身的协作实例。所有决策链、TASK 和 TODO 对社区开放。未来社区贡献者应使用自己的人机结对通过 PROACTIVE_REPORT 提交反馈，我们按协议流程吸收合并。

## 推理链

- **Zehee**: "我们会向社区公开所有的协作文件，我们作为示范，未来如果有协作者，我希望他们也用这个工具来进行主动报告（用他们自己的人机结对），我们来吸收合并。把我们的这些决策链开放给社区，让大家意识到这种写作框架的开放性和创新。"
- **Reasonix**: "好。这需要一份面向社区的 PROACTIVE_REPORT，以及一个 README 告诉访客这里是什么。协作实例本身就是一个活的示范——我们用自己的框架管理自己的进化。"

## 替代方案

1. 不开放协作实例，只推框架代码 → 与"文件即协议"的透明哲学矛盾，否决
2. 要求社区走 PR 流程而非 PROACTIVE_REPORT → 增加门槛，违反"临时参与"原则，否决
