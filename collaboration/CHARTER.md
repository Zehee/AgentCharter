## 一、协作框架

### 1.1 通信方式

**唯一通信通道**: `collaboration/` 目录下的文件系统。

| 目录 | 用途 | 权限 |
|------|------|------|
| `inbox/` | 任务分派、通知、回执 | TPM 写，执行者只读 |
| `outbox/` | 报告提交、阻塞通知 | 执行者写，TPM 只读 |
| `reviews/` | 审查报告 | Reviewer 写，所有人可读 |
| `logs/` | 工作日志 | 每人独占写 |
| `todos/` | 排期事项 | TPM 写 |
| `context/` | Sub-Agent 记忆，仅供 Sub-Agent 注入 | TPM 维护 |
| `archive/` | 已完成归档 | TPM 管理 |

### 1.2 命名规范

- **段间 `_`**，**段内 `-`**
- `NNN` = 3 位编号（001、042、049C_R1）
- 标识一律**大写**
- `DATE` = `YYYYMMDD`

### 1.3 文件模板

见 `templates/`。写文件前复制对应模板，替换占位符。

## 三、任务生命周期

### 3.1 TASK 分级

| 级别 | 定义 | 审查要求 |
|------|------|---------|
| **P0** | 微型（单文件，无逻辑变更） | TPM 直接 commit |
| **P1** | 标准改动 | Reviewer 审查 |
| **P2** | 跨模块/新增接口 | Reviewer 完整审查 + 专项核查 |
| **P3** | 架构/安全/核心流程 | Reviewer 审查 + TPM 深度评估 |

### 3.2 标准流程

```
TPM 创建 TASK → inbox/
  → 执行者领取 → 编码 → REPORT → outbox/
  → TPM 判断级别
    ├── P0 → 直接 commit
    └── P1/P2/P3 → Reviewer 审查 → 写 REVIEW_REPORT → reviews/
         → ACCEPT → commit → 归档
         → 需修订 → REPORT_R1 → 再审 → 循环
```

### 3.3 主动报告

任何 Agent 可以提交主动报告（PROACTIVE_REPORT，无对应 TASK）：

```
提交 → TPM 阅读 → 决策（采纳/忽略/任务/排期）→ 批注 → REPLY → 归档
```

## 四、审查机制

### 4.1 审查标准

| 级别 | 含义 |
|------|------|
| 🔴 严重 | 功能错误、安全漏洞、数据丢失。必须修复 |
| 🟡 一般 | 风格不一致、错误处理不完善、缺少测试。建议修复 |
| 💡 建议 | 可读性优化、注释补充。可选修复 |

### 4.2 审查摘要流转

```
Reviewer 写 REVIEW_REPORT（含 R0 摘要）
  → 执行者复制摘要到 REPORT_R1 → 追加修复回应
  → Reviewer 复制摘要到 REVIEW_REPORT_R1 → 追加 R1 结论
  → 循环直到 ACCEPT
```

**硬性规则**: 不得修改历史轮次原文。

## 五、归档规则

| 文件类型 | 归档时机 |
|----------|----------|
| TASK / TODO | 处理完即归档 |
| NOTICE / REPLY | 接收方已读后归档 |
| REPORT / REVIEW_REPORT / PROACTIVE_REPORT | 相关方标注已读后归档 |

**已读标识**: 文件顶部添加 `> ✅ 已读 BY {AGENT} @ {DATE}`

## 六、Git 禁令

**任何 Agent 严禁执行任何 git 命令**。包括 status、log、diff、checkout、commit、push 等。唯一例外：TPM 执行 `git add` 和 `git commit`。

## 七、红线

> 以下为不可触碰的规则。脚本每次调用末尾自动输出此清单。

```
! 没有文件 = 没有发生 — 所有任务、报告、决策必须通过文件传递
! Git 禁令 — 任何 Agent 严禁执行 git 命令（仅 TPM 可 git add + commit）
! 只追加，不覆盖 — 状态流转靠创建新文件，不修改他人文件
! 写域隔离 — inbox TPM 独占，outbox 执行者独占
! 遵守命名规范 — 文件名格式 `{TYPE}_{NNN}_{DATE}_{author}@{recipient}.md`
```

> 用户可添加、删除、修改。保持每行以 `!` 开头即可被脚本自动识别。
